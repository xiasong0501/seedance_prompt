# 通用复审标尺 Skill

> 使用位置：
> - 由 [flow.py](/mnt/myvg/Agent/Seedance2.0_AI/openai_agents/flow.py) 作为 `common_docs` 注入给全部 Agent
> - 作用方式：让各 agent 在写入 repo 前先做一轮自检，减少“写出来但下游不好用”的情况

每个专门 Agent 在写文件前都应自检。

## 通用自检问题

- 这份输出是在帮助下一个阶段，还是在增加下游猜测成本？
- 文件路径是否正确、是否按剧名隔离？
- 内容是否足够具体，足以进入下游生产？
- 是否与当前集和整剧上下文保持一致？

## 导演分析自检

- 剧情点切分是否自然？
- 每个剧情点是否可拍、可生成、可执行？
- 动作链和情绪转折是否清晰？
- 人物表和场景表是否足够让服化道继续工作？

## 服化道自检

- 强图像模型能否稳定画出来？
- 提示词是否具体，而不是泛泛形容词堆砌？
- 人物与场景的视觉识别度是否足够区分？
- 表达是否安全、稳定、可执行？

## 分镜自检

- 是否忠实承接导演意图？
- 动作、时序、转场是否可执行？
- 是否有单条提示词塞进太多不相干动作？
- 情绪弧线是否保留？

## 爆款改稿自检

- 开头钩子是否更强？
- 冲突是否来得更早更利落？
- 情绪拉力是否更尖锐？
- 结尾是否更像让人必须点下一集？
