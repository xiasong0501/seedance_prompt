# 题材包目录

这是当前视频理解与剧本重建阶段的推荐维护入口。

每个题材都单独放在一个目录里，目录内至少包含：

- `playbook.json`
- `skill.md`

目录命名建议直接使用中文题材名，例如：

- `先婚后爱/`
- `宅斗权谋/`
- `重生/`
- `替嫁/`

两者分工：

- `playbook.json`
  面向程序和连续性层，适合做匹配、沉淀、调试输出
- `skill.md`
  面向大模型，适合做长文本方法论注入

后续新增题材时，直接复制一个现有目录改出新题材即可。

如果整剧经验同步流程已经生成了 `__drafts__/` 下的草稿，建议再运行：

- `./run_review_genre_drafts.sh`

通过终端逐条审核后，再决定是否并入正式题材包。

如果你想直接体检某个正式题材包本身是否存在：

- 重复 / 近重复
- 错题材经验混入
- 截断文本或空泛规则
- `playbook.json` 与 `skill.md` 分工失衡

可以运行：

- `./run_genre_package_audit.sh`

这条流程支持三种模式：

- 先生成 LLM 体检报告，再逐条人工判断，最后只生成修订草稿
- 先生成 LLM 体检报告，再把你采纳的建议直接并入正式题材包
- 直接使用已有 `__drafts__/audit__<题材>__<时间戳>/` 草稿并入正式题材包

正式并入成功后，默认会自动删除对应 draft，避免重复应用。
