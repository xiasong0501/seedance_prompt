# 题材库人工审阅报告

- 日期：2026-03-26
- 范围：`skills/production/video-script-reconstruction-skill/genres/*/(playbook.json|skill.md)`
- 方法：按题材边界、专业度、字段职责、冗余度、跨题材污染与生产可复用性进行人工重构。

## 总体发现

- 多个 package 存在严重的整剧残留，混入具体角色名、具体场景锚点、某部剧专属病因/道具/人设，不适合作为题材库继续复用。
- `爱情`、`奇幻`、`玄幻`、`大女主`、`复仇逆袭` 的串题材和字段越界问题最明显，常把分镜、场景、世界观和情感规则揉成一团。
- `先婚后爱`、`宅斗权谋`、`替嫁`、`穿越`、`重生` 原本过薄，难以支撑导演、艺术与分镜多阶段共用。
- 多份文件存在空字段、重复条目、截断文本、伪结构化残留（如 `name:`、`applicable_scenes:`）以及明显不专业的模板语。

## 本次处理原则

- 删除所有具体剧残留与跨题材误吸收内容，只保留可泛化的题材经验。
- 按字段职责重写：`script_hooks` 只讲叙事发动，`director_focus` 只讲镜头执行，`scene_focus` 只讲空间组织，`dialogue_timing_rules` 只讲对白与节奏。
- 补齐空字段与薄弱题材，使每个 package 都能覆盖人物、场景、分镜、连续性与负面模式。
- `skill.md` 统一改为操作型摘要，供导演/艺术/分镜阶段直接提炼。

## 各题材处理概览

### 先婚后爱
- 原 package 偏提纲化，导演、对白、连续性与负面模式字段长期缺失。
- 本次补齐完整题材框架，并把“契约/共居/立场变化”作为核心方法论固定下来。

### 复仇逆袭
- 原 package 混入了大量奇幻/现代异病与模板化元数据，出现 name/applicable_scenes 等非成品残留。
- 本次改为纯题材方法论，保留复仇链条、公开围观与证据回收三大核心。

### 大女主
- 原 package 串入大量爱情、真千金、玄幻残留，题材边界模糊。
- 本次重写后把“主体性、能力、代价、权力重组”作为唯一中心。

### 奇幻
- 原 package 存在严重的具体剧污染，混入人物姓名、病房调度、现代爽文场面等。
- 本次重建后将奇幻与玄幻切开：奇幻强调“异常侵入日常”，不再承担修炼体系职责。

### 宅斗权谋
- 原 package 过于简陋，缺少人物、对白、连续性、负面模式等核心维度。
- 本次补足礼制、主位、证据链、称呼系统等宅斗权谋必要骨架。

### 替嫁
- 原 package 只有最粗略提纲，无法支撑生产级拆解。
- 本次补上替嫁题材独有的身份压力、仪式压力与试探结构。

### 爱情
- 原 package 受到最严重的串题材污染，混入玄幻、现代神医、豪门打脸、具体剧角色名等大量残留。
- 本次彻底清空非爱情经验，把内容收回到“关系推进、情感节奏、亲密距离、障碍设计”四条主轴。

### 玄幻
- 原 package 与奇幻、爱情、高位打脸短剧混杂严重，字段职责混乱且重复密集。
- 本次明确玄幻边界：保留修炼/血脉/法器/宗门/禁制逻辑，移除现实爽文与爱情残留。

### 穿越
- 原 package 过薄，只覆盖了最表层的错位感，没有把身份与规则压力写出来。
- 本次补足穿越题材的证据链、隐藏链和利用链。

### 萌宝团宠
- 原 package 有一定基础，但缺少连续性、负面模式和完整分工。
- 本次保留“孩子推动主线”这一正确方向，并把它收束为更稳定的题材规则。

### 重生
- 原 package 偏薄，能看到方向，但缺少连续性、场面与操作层规则。
- 本次强化了重生题材最重要的三件事：前世账、今生局、代价感。
