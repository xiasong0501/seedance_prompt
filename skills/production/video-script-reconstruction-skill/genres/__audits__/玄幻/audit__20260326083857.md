# 题材包体检报告：玄幻

- 健康分：42
- 综合结论：该“玄幻”题材包目前更像多个具体短剧样本经验的拼接集合，而不是稳定、可复用的玄幻题材规则库。最大问题不是单条质量都差，而是：1）跨文件原文重复严重；2）大量规则明显带有具体剧集残留（申家、周煜、玉珏残片、钢饼、闻浅/文青、医院病房、豪宅客厅等）；3）不少条目更像爱情/打脸逆袭/现代医疗悬疑/宅门奇幻的子类型经验，而非“玄幻”母题共性；4）playbook 与 skill 职责混杂，结构化字段里塞入了大量镜头口令、具体场面调度和样本连续性约束。优点是已有一些可保留的高价值方向，如“玄幻感优先通过人物异质性而非廉价特效”“现代真实空间嵌入轻异质设定”“超常能力需绑定人物处境与空间留驻理由”。但需要一次较强的去样本化、去重复、重分工整理。

## 预检摘要

- playbook 条目数：154
- skill 规则数：140
- 包内疑似重复：12
- 跨题材高重合：4
- 可疑条目：0

## 亮点

- 已形成少量可泛化的玄幻表达原则，如“人物异质造型优先于廉价特效”。
- 能意识到玄幻在短剧中常与现代空间、关系推进、打脸结构结合，具备一定生产实感。
- 部分规则具备可执行性，如卡点、反应镜头、能力兑现方式等。
- 已出现少量真正贴近题材核心的规则：超常能力要与人物处境、空间绑定，而非只做打架工具。

## 发现的问题

### F01 [high] duplicate

- 位置：playbook.json 与 skill.md 多处逐字重复
- 问题：两文件存在大量 1.0 相似度的逐字重复条目，如“女主若要快速推进关系，可借外部威胁触发‘假身份公开认领’装置”“用‘公开纠缠场’承接男女主关系成立，比私下告白更高效”“爱情戏的情绪支付点常来自一方公开命名关系、另一方默认认领”“集尾优先停在‘决定已做出、方法未展示’的节点”等。
- 影响：playbook 应承载结构化、可检索规则；skill 应承载解释性方法论。原文重复会放大噪音，降低检索效率，也让后续维护时难以判断以哪一处为准。
- 建议：保留 playbook 中更短、更结构化的版本；skill 仅保留少量需要展开解释的规则，并改写为方法论段落，避免逐条照抄。
- 置信度：0.99

### F02 [high] cross_genre_overlap

- 位置：playbook.json::script_hooks / storyboard_focus / dialogue_timing_rules；skill.md 多章节
- 问题：大量核心条目更像爱情短剧、打脸逆袭、爽剧卡点通用规则，而非玄幻专属规则，例如“公开认领”“关系宣言停顿”“男主高光先靠不惧权势的嘴上压制”“反派被打脸后给反应近景”“打脸对白宜短句狠句少解释”等。
- 影响：如果题材包主体由跨题材通用爽点规则构成，模型会把“玄幻”误学成“现代爱情+打脸逆袭+一点超常元素”，削弱题材辨识度。
- 建议：将明显通用的爽剧/爱情规则降权、删减，或改写为“当玄幻与爱情/逆袭混搭时可用”的条件化规则；优先补足世界观、修行体系、代价机制、异能显影、禁忌规则等玄幻核心能力。
- 置信度：0.96

### F03 [high] scope

- 位置：playbook.json::continuity_guardrails；skill.md::生产红线
- 问题：多条连续性规则明显是具体剧集残留，而非题材级规则，如“让申家成为顶级权贵”“豪宅客厅、盲杖、墨镜、挂钟、雪糕”“恢复视力”“神经系统无反应”“中毒”“蛊虫”“拜帖数量（八+一）”“周煜”“玉珏残片”“钢饼”“闻浅/文青”等。
- 影响：题材包应服务多个项目复用。具体角色名、道具名、数字锚点、单集剧情节点会严重污染泛化能力，甚至误导模型在新项目中凭空生成不相关设定。
- 建议：这些条目原则上从题材包删除；若其背后有可泛化经验，应抽象改写，例如把“玉珏残片必须延续至第3集并首次展现功能”改成“首集埋下的核心灵异道具应在前3集内首次兑现功能，且外观连续性保持一致”。
- 置信度：0.99

### F04 [high] cross_genre_overlap

- 位置：playbook.json::scene_focus / costume_focus / director_focus；skill.md::场景与调度、服化道统一
- 问题：医院病房、会诊戏、年轻医护、白大褂、CT片展示位、病床/隔帘/显示屏等规则占比很高，明显偏向现代医疗悬疑/异病题材，而非玄幻母题本身。
- 影响：这会把“玄幻”错误收窄为“现代医院里出现蛊虫/异病”的特定子样本，导致其他玄幻场景（宗门、秘境、宅门、朝堂、异界、山野）能力缺失。
- 建议：将医疗异病相关内容收束为少量条件化规则，例如“现代玄幻子型中，可在理性空间嵌入非常规病因”；删除过细的病房调度与制服规则，或迁移到更具体子题材包。
- 置信度：0.97

### F09 [high] missing

- 位置：整体包体
- 问题：缺少玄幻题材最关键的母题能力：世界规则建立、能力来源与代价、修行/血脉/器物体系、禁忌与反噬、凡俗秩序与超常秩序冲突、设定显影节奏、术法命名与信息密度控制。
- 影响：当前包体虽然有“轻玄异病因”“人物异质造型”等局部经验，但没有回答“玄幻为什么成立、如何持续成立、如何区别于奇幻/悬疑/爱情爽剧”。
- 建议：新增少量高质量母规则，优先覆盖：能力必须有代价；超常规则首次出现要同时给出边界；核心器物/血脉/术法需在前几集完成一次可验证兑现；世界观解释遵循‘现象—误判—局部验证—更大谜团’。
- 置信度：0.98

### F05 [medium] duplicate

- 位置：playbook.json 多字段内部近重复
- 问题：同义或近义条目过多，如“集尾卡点优先停在‘后果将至、解释未给’的瞬间”在 script_hooks、storyboard_focus、skill 分镜中反复出现；“动作戏重结果不重过程/一招制敌/起手—命中—反应”高度重叠；“妆发必须服务近景表情读取”在 costume_focus 与 skill 中多次重复；“高位女性角色的神性通过‘动作克制+眼神穿透’建立”重复两次。
- 影响：近重复会稀释真正有效的规则密度，也让模型误以为某些非玄幻通用拍法是题材核心。
- 建议：每组只保留一条最完整、最可执行的版本。优先保留信息量更高、约束更清晰、适用面更广的条目。
- 置信度：0.95

### F06 [medium] structure

- 位置：playbook.json::script_hooks / director_focus / dialogue_timing_rules / scene_focus
- 问题：字段职责混乱。script_hooks 中混入视觉站位、造型优先级、宅门空间逻辑；dialogue_timing_rules 中混入导演/剪辑规则如“传播型金句段应减少切镜频率”；scene_focus 中混入镜头动作“权力反转时必须拍摄身体靠近/转身面向”；director_focus 中混入过细模板化镜头语言。
- 影响：字段职责不清会降低包体可维护性，也不利于后续程序化调用。
- 建议：按“叙事钩子/人物与世界/场景/分镜/对白/连续性”重新归位：叙事结构留在 script_hooks，镜头与节奏留在 storyboard/director，空间约束留在 scene_focus。
- 置信度：0.94

### F07 [medium] reasonableness

- 位置：playbook.json::scene_focus, director_focus；skill.md::场景与调度、分镜表达
- 问题：部分规则过细到接近单次拍摄口令，如“求援者距主角1.2米，质疑者距主角1.8米”“结尾卡点：侧脸特写占2/3画面 + 眼神骤冷 + 字幕淡入 + 停帧2秒”“0.3–0.5s黑屏/呼吸停顿”。
- 影响：题材包应提供稳定原则，而不是把某次有效镜头方案硬编码为普适规则。过细参数会降低适配性，也容易在不同项目中显得机械。
- 建议：改写为区间化、原则化表达，例如“跪者、主角、质疑者需形成清晰高低位与压迫梯度”“关键认知冲击后留短暂停顿以便观众消化”。
- 置信度：0.92

### F08 [medium] clarity

- 位置：playbook.json::negative_patterns / dialogue_timing_rules / storyboard_focus
- 问题：存在空泛或口号化表达，如“高效悬念对白以短句、狠句、离谱句为主”“冲突升级时要让每句关键台词都推动局势”“不要在分镜上过度追求花哨运镜”。这些方向没错，但缺少玄幻语境下的判断标准。
- 影响：空泛规则难以指导生成，也难以人工判断是否执行到位。
- 建议：补充适用条件或可观察标准，例如“当超常设定首次显影时，台词应优先制造认知冲突而非解释术语”。
- 置信度：0.84

### F10 [medium] cross_genre_overlap

- 位置：playbook.json::script_hooks, character_focus, costume_focus
- 问题：“奇幻宅门题材”“婚内背叛清算戏”“背叛型伴侣”“第三者角色”“高位男性西装”等条目把包体拉向宅斗/婚恋/现代豪门短剧。
- 影响：这些可以作为玄幻混搭子型经验，但不应占据玄幻母包的大量篇幅，否则会误导为‘玄幻=现代豪门背叛+一点异能’。
- 建议：保留少量能泛化的混搭规则，其余迁移到更具体子题材，或改写为条件句：“当玄幻与现代豪门/婚恋混搭时……”。
- 置信度：0.93

### F11 [medium] scope

- 位置：playbook.json::continuity_guardrails
- 问题：连续性护栏中既有题材级规则，也有单项目剧情推进要求，如“不可在第2集结尾解除”“必须延续至第3集并首次展现功能”“3集内澄清”。
- 影响：题材包不应绑定固定集数节奏到具体道具和身份错位，除非抽象成通用前几集节奏原则。
- 建议：将具体集数要求抽象为“前3集内”“早期篇章内”且去掉专名；若无法抽象则删除。
- 置信度：0.9

### F12 [low] missing

- 位置：playbook.json 空字段
- 问题：core_audience_promises、character_design_focus、scene_design_focus 为空。
- 影响：空字段本身不是致命问题，但在当前包体已经高度混乱的情况下，说明应有的高层概括缺失，导致大量细碎规则无上位统摄。
- 建议：不要机械补满；优先补 2-4 条高层承诺/焦点，用来约束下层规则，例如“玄幻承诺：凡俗秩序中出现可验证的超常规则，并持续升级其代价与后果”。
- 置信度：0.88
