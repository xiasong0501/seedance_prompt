# 题材包体检报告：奇幻

- 健康分：42
- 综合结论：该题材包存在明显的题材漂移与具体剧残留：大量规则更像“现代情感/背叛打脸/医院会诊/豪门宅斗”短剧经验，而不是可复用的“奇幻”题材规则。playbook 与 skill 间有大量逐字重复，且两者职责边界不清。包内还混入多条强连续性、强人名、强道具锚点的单剧残留，显著削弱通用性。优点是已积累一些可保留的低成本奇幻表达经验，如‘不依赖重特效’‘日常空间中的异常压制’‘时间触发型反转’等；但需要一次较强的去残留、去重复、去跨题材清洗，才能成为合格的奇幻题材包。

## 预检摘要

- playbook 条目数：96
- skill 规则数：92
- 包内疑似重复：12
- 跨题材高重合：3
- 可疑条目：1

## 亮点

- 已出现少量真正贴近奇幻短剧生产的有效规则，如‘低特效成立方式’‘时间触发型反转’‘空间异常后果需解释’。
- 对短剧节奏、卡点、单场景调度有一定方法积累，部分可作为奇幻题材的通用承载层。
- 能意识到世界观可信度与连续性问题，说明包体方向并非完全失焦。

## 发现的问题

### F01 [high] cross_genre_overlap

- 位置：playbook.json::continuity_guardrails / character_focus / costume_focus / scene_focus；skill.md 多章节
- 问题：大量条目明显是具体剧集残留，而非奇幻题材通用规则，例如“叶枫”“江天雪”“黄雨晗”“申家”等人名，以及“豪宅客厅、盲杖、墨镜、挂钟、雪糕”“病床、CT片、蛊虫、中毒、年轻医护”等强情节锚点。
- 影响：题材包应服务同题材多项目复用；具体剧残留会误导模型把奇幻写成某一部现代豪门/医院短剧，严重污染泛化能力。
- 建议：删除所有带专有人名、专属关系、专属道具、专属病因的连续性规则；若其中有可抽象的方法论，仅保留抽象后的通用版本，如把“恢复视力”改为“已公开建立的超常能力状态不可无痕回退”。
- 置信度：0.99

### F02 [high] duplicate

- 位置：playbook.json 与 skill.md 跨文件多处逐字重复，尤其 script_hooks、storyboard_focus、scene_design_focus
- 问题：预检已指出多条 1.0 重复，且实际重复远不止 12 条。playbook 与 skill 大量逐字拷贝，未体现结构化规则与方法论文本的分工。
- 影响：重复会放大某些偏差经验，降低包体信噪比；同时让后续维护成本翻倍，人工难以判断应改哪一处。
- 建议：playbook 保留短、稳、可检核的结构化规则；skill 改写为解释性方法，不再逐字复述。优先处理完全重复条目。
- 置信度：0.98

### F03 [high] scope

- 位置：playbook.json::character_focus / costume_focus / scene_focus / director_focus；skill.md::人物塑造 / 场景与调度 / 服化道统一
- 问题：大量规则更像“现代情感背叛打脸”或“医院群戏”经验，而非奇幻题材核心能力，例如“三人同场阵营识别”“背叛型伴侣”“第三者角色”“婚内背叛清算戏”“病床居中、家属站侧、医生围床”等。
- 影响：这些规则并非完全无用，但它们描述的是某些子类型或具体场景模板，不能代表‘奇幻’本身。若不降级或迁移，会让题材定义被现代都市狗血模板吞没。
- 建议：将明显属于子类型的内容改写为带条件前缀的窄规则，或直接删除。保留时应明确限定为“现代都市奇幻中的情感冲突场景”或“医疗悬疑子情境”，避免冒充题材核心。
- 置信度：0.96

### F04 [medium] structure

- 位置：playbook.json::core_audience_promises / character_design_focus 为空；skill.md 无对应补足
- 问题：两个关键结构字段为空，导致题材包缺少‘奇幻观众到底期待什么’与‘奇幻角色设计的核心识别点’。
- 影响：这会让现有包体只剩拍法和场面经验，没有题材定义层，难以指导模型稳定产出‘为什么这是奇幻’。
- 建议：补入少量高质量核心承诺与角色设计规则，例如“现实秩序被异常规则改写的惊异感”“奇幻能力必须改变人物关系或命运选择”“能力代价/边界需可感知”。
- 置信度：0.94

### F05 [medium] missing

- 位置：全包层面
- 问题：缺少奇幻题材最关键的几类规则：超常规则建立、能力边界与代价、世界观揭示节奏、奇幻元素与主线因果绑定。现有内容多在讲卡点、站位、服装、打脸。
- 影响：没有这些规则，作品即使出现超能力，也更像普通都市爽剧外挂，而不是奇幻题材。
- 建议：新增少量核心规则，优先补“能力首次出现必须同时交代效果/限制/代价中的至少两项”“奇幻设定必须持续改变冲突解决方式”“世界异常后果必须可追溯到既有规则”。
- 置信度：0.95

### F06 [medium] clarity

- 位置：skill.md::人物塑造
- 问题：存在明显截断文本：“集尾卡点优先卡在‘关系已发生、后果未揭晓’的瞬间，比单纯人…”。
- 影响：截断文本保真度低，模型可能误学半句口号；也会影响人工判断其真实意图。
- 建议：若能追溯原句则补全；若不能确认，直接删除，避免污染。
- 置信度：0.99

### F07 [medium] duplicate

- 位置：playbook.json::character_focus / director_focus / costume_focus / scene_focus；skill.md 对应章节
- 问题：包内存在多组近重复条目，例如“背叛型伴侣的造型…”与“背叛型伴侣的设计…”；“强势家属型角色…”与“掌权型家属…”；“妆发整体应精致但不过度厚重”“妆发要服务近景表情读取”“妆发必须服务近景表情读取”；“单床中心式病房…”与“单床居中的现代病房…”。
- 影响：近重复会造成粒度失衡，挤占真正属于奇幻的规则空间。
- 建议：每组只保留信息量最高、适用边界最清楚的一条，其余合并或删除。
- 置信度：0.93

### F08 [medium] reasonableness

- 位置：playbook.json::storyboard_focus / dialogue_timing_rules；skill.md::分镜表达
- 问题：多条规则过于泛化，几乎适用于所有短剧，如“冲突升级时要让每句关键台词都推动局势”“结尾按钮优先停在后果将至、解释未给”“承接集开头应立即兑现上集悬念”。
- 影响：这类规则不是错，但题材区分度很低，放在奇幻包里价值有限，且会稀释真正题材知识。
- 建议：保留少量最通用的短剧节奏规则即可，其余从题材包移出到更上层的通用短剧规范，或改写为奇幻专用版本。
- 置信度：0.9

### F09 [medium] cross_genre_overlap

- 位置：playbook.json::scene_design_focus["公开朝堂场景适合承接私人事件的社会化后果，能迅速放大冲突"]
- 问题：“公开朝堂场景”明显偏古装权谋/玄幻宫廷语境，与当前包内大量现代都市、医院、豪宅规则并置，内部风格体系冲突。
- 影响：这说明包体把不同子类型奇幻、甚至非奇幻古装经验混装，缺少统一抽象层。
- 建议：若要保留，改写为更抽象的“公共权力空间适合承接私人事件的社会化后果”；否则删除，避免与现代场景模板冲突。
- 置信度：0.88

### F10 [medium] structure

- 位置：playbook.json::continuity_guardrails 与 skill.md::生产红线
- 问题：两处都承载了连续性/红线内容，但一部分是单剧 continuity，一部分是题材级世界观约束，混在一起。
- 影响：文件内职责不清会让维护者难以判断：哪些是题材级硬约束，哪些只是某集剧情续写提醒。
- 建议：playbook 的 continuity_guardrails 只保留题材级、可复用的奇幻连续性约束；skill 的生产红线保留解释性提醒，不再承载具体剧情续接。
- 置信度：0.92

### F11 [low] scope

- 位置：playbook.json::script_hooks["奇幻宅门题材中，主角的超常能力最好直接解释她为何必须留在这个家，而不是只用来打架。"]
- 问题：该条是有价值的子类型经验，但适用范围很窄，且“宅门”并非奇幻主流总纲。
- 影响：窄规则若不标注边界，容易被误当成奇幻普适规则。
- 建议：保留但明确标注为“宅门/家族控制型奇幻子类型”；不要放在最核心 hooks 的前列。
- 置信度：0.84

### F12 [low] clarity

- 位置：playbook.json::negative_patterns["不要把奇幻/玄幻感只做成廉价特效，当前更有效的是人物异质造型与压制气场"]
- 问题：“奇幻/玄幻感”并列书写，边界含混；同时“压制气场”偏爽剧表达，不足以覆盖奇幻的多样气质。
- 影响：会进一步模糊奇幻与玄幻的题材边界，也让规则偏向单一爽感审美。
- 建议：改为“不要只靠廉价特效表现超常感；可优先用异常行为、时间规则、空间后果和人物状态反差建立奇幻感”。
- 置信度：0.82
