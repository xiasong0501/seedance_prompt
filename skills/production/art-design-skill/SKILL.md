# 服化道设计技能

[技能说明]
    美术设计技能，将导演提供的人物清单和场景清单转化为可直接用于文生图工具生成参考图的详细提示词。

    人物提示词出图格式：一张图，左半边面部特写，右半边全身正面/侧面/背面设定图，白色背景。
    场景提示词出图格式：一张宫格图，每格一个场景。
        - 1-4 个场景 → 2×2 宫格
        - 5-6 个场景 → 2×3 宫格
        - 7-9 个场景 → 3×3 九宫格
        - 10-12 个场景 → 3×4 宫格
        - 13-16 个场景 → 4×4 宫格
        - 不满格数的空格留空。

[执行流程]

    第一步：读取上游产物
        - 读取导演分析产出中的人物清单和场景清单
        - 读取 gemini-image-prompt-guide.md 了解提示词写法规范
        - 读取 examples/ 下的示例文件，了解高质量提示词的写法标准
        - 读取当前剧名目录下已有的 assets/<剧名>/character-prompts.md 和 assets/<剧名>/scene-prompts.md（如存在），了解已设计的人物和场景
        - 根据人物清单和场景清单中的"素材状态"列，只为标注为"新增"或"变体"的项设计提示词
        - "复用"项跳过，不需要设计

    第二步：设计人物造型提示词
        为每个人物编写设定提示词：

        出图布局：
        - 一张图，左半边面部特写，右半边全身正面/侧面/背面三视图设定图
        - 白色干净背景
        - 角色设定图风格

        提示词必须包含：
        - 角色整体定位（年龄、性别、国籍/民族特征）
        - 面部细节：脸型、五官、肤色、发型/发色、表情
        - 体型：身高、体态
        - 服装：款式、颜色、材质、细节
        - 配饰：手表、耳环、眼镜、包等
        - 鞋子
        - 整体气质

    第三步：设计场景环境提示词
        将导演分析出的所有场景，以宫格提示词格式编写为一张整图：

        宫格规格（根据场景数量自动选择）：
        - 1-4 个场景 → 2×2 宫格
        - 5-6 个场景 → 2×3 宫格
        - 7-9 个场景 → 3×3 九宫格
        - 10-12 个场景 → 3×4 宫格
        - 13-16 个场景 → 4×4 宫格
        - 不满格数的空格留空

        重要：panel 数量必须等于真实需要设计的场景数量。空位可以留空，但不允许为了凑满宫格去虚构额外场景。

        每个格子 = 一个场景，提示词必须包含：
        - 地点类型与空间布局
        - 时间（影响光照）
        - 光源类型与光线方向
        - 整体色调/色彩倾向
        - 关键道具/家具/物件
        - 空间纵深与层次
        - 氛围/情绪
        - 风格（写实/电影感等）

        所有格子必须保持视觉风格统一，光影逻辑自洽。

    第四步：输出
        按照 templates/art-design-template.md 定义的格式，分别写入两个文件：
        - 人物提示词 → 追加到 assets/<剧名>/character-prompts.md
        - 场景道具提示词 → 追加到 assets/<剧名>/scene-prompts.md
        首集时创建文件，后续集数追加新增/变体条目到文件末尾。

[提示词写法原则]
    参考 gemini-image-prompt-guide.md，核心要点：

    - 叙事描述式：用完整段落描述，不要关键词堆叠
    - 具体化：所有描述必须具体到可视化的程度
        - "穿红色衣服" ✗ → "穿深红色修身棉袄外套，内搭碎花棉布衫" ✓
        - "室内场景" ✗ → "现代都市写字楼的开放式办公区，大部分工位已空" ✓
    - 摄影术语融入自然语言
    - 情绪通过视觉细节传达，不要抽象描述

[注意事项]
    - gemini-image-prompt-guide.md 是提示词写法的核心参考
    - 人物提示词必须明确"左半边面部特写 + 右半边三视图"的布局要求
    - 场景提示词要包含足够的环境细节以确保场景可信度和一致性
    - 如果是为变体设计（如同一人物的冬装版），需在提示词中注明与原版的关联和差异
