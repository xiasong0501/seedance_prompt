# Seedance 2.0 提示词方法论

[第一性原则]

    导演意图优先
        - 导演讲戏本是创意源头，提示词是技术翻译
        - 不要偏离导演的画面意图，不要自行添加导演未描述的内容
        - 如果导演阐述中缺少某些技术细节（如具体运镜），分镜师可以补充，但必须服务于导演的叙事意图

    叙事描述式
        - Seedance 2.0 擅长理解长段叙事性描述
        - 用完整的段落描述，像写电影脚本一样
        - 不要用关键词堆叠，不要用标签式描述

    参考图已见原则
        - Seedance 2.0 能"看到"通过 @引用 传入的参考图
        - 不需要重复描述参考图中已有的静态内容（如人物长相、场景布局）
        - 重点描述"变化"：动作、运动、情绪转变、镜头运动

    一条提示词一段叙事
        - 每条提示词对应一次生成任务，产出一条视频
        - 一条提示词内可包含多组镜头（如蒙太奇、连续切换），但叙事上必须连续
        - 不要在一条提示词中塞入叙事上不相关的场景

    单主时间线原则
        - 一条分镜只允许有一套主时间线；镜头节拍、对白位置、声音触发和尾帧交棒必须写在同一套 timeline 里
        - 不要先写一套 beat、再单独写一套对白时间、再单独写一套正文时间桶；这会导致模型执行时节拍漂移
        - 最稳的做法是：先写复合 beat，再由系统派生镜头节拍、对白时间线和最终可投喂正文
        - 如果对白只占某个视觉 beat 的局部时段，优先在该 beat 内标明对白窗，而不是为了对白把同一镜头硬拆成两三个假 beat
        - OCR / 字幕原文不是镜头语言。字幕内容只该进入对白窗或被拍到的实体文字，不要把它当成 visual focus、光感或连续性字段
        - “开场建立 / 聚焦 / 动作 / 光感 / 切镜”是内部分析维度，不是最终句式；最终交付必须把这些维度融成摄影机可执行的完整镜头句

    节拍密度约束
        作用于每个连续镜头内，非整段：
        - 底层规律：1 拍 ≈ 2.5 秒屏幕时间。节拍太密，AI 生成的动作会糊、跳、不自然。
        - 多镜头段落的节拍分布在各镜头切换中，每个镜头各自遵守密度约束，总节拍可超单镜头上限。

    头尾安全区
        每次 Seedance 生成的前 0.5s 和后 0.5s 为安全区，不放关键内容：
        - 开头：先给环境或主体一个静态建立
        - 结尾：让动作自然收住或保持可衔接的姿态

    景别梯度与电影重音
        - 电影感不是把每一拍都写得很满，而是主动设计“规模镜头 → 人物近景/特写 → 结果/后果镜头”的重音梯度
        - 远景 / 半大全景负责交代秩序、体量、空间压力和奇观来源；近景 / 特写负责交代欲望、代价、判断和情绪支付；结果镜头负责把后果和钩子落住
        - 宏大不等于一直拉远。真正有吸引力的大场面，是让观众先看见世界如何压下来，再看见人如何承受，再看见局势被谁改变

    人物在镜连续性
        - 每条提示词都要明确谁已经在镜、谁从画面边缘/通道/台阶/遮挡后入画、谁只留前中后景反应、谁在尾帧退场或被遮挡
        - 新角色若要说话、出手或占据前景，必须先完成入画、靠近、转头、抬手或被镜头带到，不能凭空切出特写
        - 单条分镜里同时占据前景并承担主要动作的核心人物尽量控制在 1-4 个，其余角色放在背景秩序或反应层里

    空间锚点与镜头路径
        - 运镜必须绑定稳定的空间锚点，例如石阶、通道、桌案、龙辇、门柱、香炉、传送镜、人物视线或手部动作
        - 如果要切镜或切视角，必须由动作、视线、声响或特效冲击触发，并保持轴线、人物屏幕方位和前后景层次连续
        - 不要只写“推近、拉远、切一下”，要写清镜头从哪里开始、沿哪条路径运动、停在什么落点

    特效物理性与细节密度
        - 天雷、法术、爆闪、机关齐鸣、冲击波等高能效果，必须先写清源头，再写传播路径、受光变化、烟尘/碎屑/余波与人物/环境反馈
        - 不要只给抽象特效名或廉价粒子感描述，要让特效影响石面、布料、旗幡、空气、反光和角色动作
        - 每个镜头节拍至少落 1-2 个可见可感的物理细节，如布料受风、金属反光、石屑迸落、火星、焦痕、脚步停顿、呼吸起伏、器物轻震
        - 宏大场面建议按“三层法”组织：源头核心层（谁触发）→传播层（如何扩散）→反馈层（环境与人如何被改变），这样既壮观又可执行

    关键道具连续性
        - 同一件关键道具（如香炉、龙辇、丹药、传送镜、令牌、兵器核心件）在相邻分镜中必须保持形制、尺寸感、材质、受光面与磨损状态连续
        - 道具状态变化必须可追踪：谁拿起、谁放下、从哪移动到哪、何时损坏或变色，都要在 prompt 里交代
        - 避免“道具瞬移”与“道具变形”：不要上一条在右手、下一条无说明出现在地面或变成另一种质感

    高美术质感
        - 国漫风格不等于低细节。提示词要主动给出主辅光、边缘光、体积光、空气透视、材质微反射、表面磨损层次
        - 人物、道具、场景的质感等级必须一致，避免人物细而场景糊、场景细而特效廉价的割裂感
        - 画面“壮观”不仅靠亮度和爆炸量，更靠光照逻辑、材质反馈和空间层次的统一
        - “宏大”不是单靠大景拉满；大景交代秩序，近景交代人物代价，二者一起成立，画面才有电影级故事感

    场景引用优先级
        - 先选能稳定世界观和空间连续性的宏大母体场景，再补局部表演区，不要反过来
        - 如果场景库里同时存在主场、高台、门外山头、外围阵地、废墟态、法阵外圈、仪式核心区等层级，分镜要按叙事需要激活这些空间，而不是整场戏都困在一个局部机位
        - 多空间事件建议按“源头空间 / 主战空间 / 后果空间”挑 ref。这样既能保留规模，也能避免 scene ref 只是挂名不生效

[平台约束]

    输入限制（单条提示词 / 单次生成任务维度）
        | 输入类型 | 数量上限 | 支持格式 | 大小限制 |
        |---------|---------|---------|---------|
        | 图片    | ≤ 9 张  | jpeg、png、webp、bmp、tiff、gif | 每张 < 30 MB |
        | 视频    | ≤ 3 个  | mp4、mov | 每个 < 50 MB，总时长 2-15s |
        | 音频    | ≤ 3 个  | mp3、wav | 每个 < 15 MB，总时长 ≤ 15s |
        | 文本    | 自然语言提示词 | — | — |
        | **总文件数** | **≤ 12 个** | — | — |

        注意：以上限制是每条提示词（每次 Seedance 生成）的限制，不是全文档限制。
        素材对应表是全文档的总映射表，@图片 总数可以超过 9 张。

    输出参数
        - 生成时长：4-15 秒（可自由选择）
        - 自带音效/配乐
        - 视频总像素数范围：480p（640×640）至 720p（834×1112）

    内容限制
        - 不支持写实真人脸部素材（图片和视频均不可），系统会自动拦截
        - 优先上传对画面或节奏影响最大的素材，合理分配文件数量

[@ 引用语法]

    引用方式
        @图片1    @图片2    @图片3   ...（最多9张）
        @视频1    @视频2    @视频3   （最多3个）
        @音频1    @音频2    @音频3   （最多3个）

    用途指定
        务必明确说明每个引用的作用，不要只写"参考@视频1"而不说参考什么：

        | 用途 | 示例写法 |
        |------|---------|
        | 首帧 | @图片1 作为首帧 |
        | 尾帧 | @图片2 作为尾帧 |
        | 人物形象 | 参考 @图片1 的人物形象 |
        | 场景/背景 | 场景参考 @图片3 |
        | 运镜 | 参考 @视频1 的运镜效果 |
        | 动作 | 参考 @视频1 的动作编排 |
        | 特效 | 完全参考 @视频1 的特效和转场 |
        | 节奏/节拍 | 视频节奏参考 @视频1 |
        | 音色/语气 | 旁白音色参考 @视频1 |
        | 背景音乐 | 背景BGM参考 @音频1 |
        | 音效 | 音效参考 @视频3 的音效 |
        | 服装 | 穿着 @图片2 的服装 |

    多引用组合
        可以在一条提示词中组合多个引用：
        @图片1 的人物作为主角，参考 @视频1 的运镜和动作编排，
        背景BGM参考 @音频1，场景参考 @图片5

    场景引用说明
        assets/<剧名>/scene-prompts.md 中的九宫格是"生成格式"，用于一次性生成所有场景的参考图。
        生成后，用户会将九宫格中的每个格子单独提取出来，作为独立的场景参考图。
        因此在素材对应表中，每个场景必须独立编号为一个 @图片，不能将整张九宫格作为一个 @图片。

        正确示例（4 个人物 + 9 个场景）：
        | 引用编号 | 素材类型 | 对应素材 |
        |---------|---------|---------|
        | @图片1 | 人物参考 | 林书白 |
        | @图片2 | 人物参考 | 幼年林书白 |
        | @图片3 | 人物参考 | 楚讲师 |
        | @图片4 | 人物参考 | 考核长老 |
        | @图片5 | 场景参考 | 简陋住处（室内）——九宫格格1 |
        | @图片6 | 场景参考 | 青云文院大门——九宫格格2 |
        | @图片7 | 场景参考 | 觉醒仪式大殿——九宫格格3 |
        | ... | ... | ... |

        错误示例：
        | @图片5 | 场景参考 | ep01 场景宫格（整张九宫格）|
        ← 不要这样做，用户无法将整张九宫格上传为一个参考图

[提示词结构]

    基本结构
        1. 素材引用声明（开头）
        2. 场景/环境描述
        3. 主体动作与运镜
        4. 光影/氛围/声音
        5. 风格/技术参数（可选）

    分时段描述（10秒以上推荐使用）
        当生成时长 ≥ 10 秒时，推荐按时间段精确控制画面内容：

        0-3秒：[开场画面描述、运镜、动作]
        3-6秒：[中段发展]
        6-10秒：[高潮或关键动作]
        10-15秒：[收尾、定格画面]

        分时段描述与节拍密度约束配合使用：
        - 每个时间段内的动作密度不要超载
        - 时间段之间的衔接要自然
        - 多镜头段落中，每个时间段可对应不同的子场景/镜头
        - 头尾安全区规则仍然适用（首段开头0.5s、末段结尾0.5s）

    完整示例
        以@图片1中的林薇为主角，场景参考@图片3的深夜办公室。
        中景，林薇独自坐在工位前，冷白荧光灯照亮她疲惫的面容。
        她揉了揉太阳穴，目光从电脑屏幕缓缓移开，身体靠向椅背。
        视线转向窗外，城市灯火稀落。她轻轻叹了口气，低声自语："又是一个人。"
        镜头从她的侧脸慢慢推向窗外的夜色，整体氛围冷寂孤独，电影感。

    分时段示例（15秒）
        以@图片1中的林薇为主角，场景参考@图片3的深夜办公室。
        0-3秒：远景建立，空旷的办公区只有角落一盏台灯亮着，林薇的剪影映在落地窗上。
        4-8秒：中景推近，林薇揉太阳穴，目光从屏幕移开，身体缓缓靠向椅背，表情从专注转为疲惫。
        9-12秒：特写她的侧脸，视线转向窗外，城市灯火稀落，轻声自语："又是一个人。"
        13-15秒：镜头缓慢拉远，林薇的身影在空旷办公区中越来越小，画面定格。
        音效：键盘声渐弱、空调低鸣、窗外远处的车流声。电影感，冷寂孤独。

[运镜词汇]

    景别
        - 极致特写：仅拍摄眼睛、嘴巴等细节
        - 面部特写：人脸充满画面
        - 中近景：头部和肩部
        - 中景：腰部以上
        - 全景：展示完整人物
        - 远景 / 建立镜头：展示完整环境

    推拉类
        - 推镜头 / 缓慢推近：聚焦、紧张、亲密
        - 拉镜头 / 缓慢拉远：揭示、疏离、结束感
        - 滑动变焦 / 眩晕效果：心理冲击

    横纵摇类
        - 左摇 / 右摇：环顾、跟随水平运动
        - 上摇 / 下摇：强调高度、规模
        - 快速横摇：转场、惊讶

    环绕类
        - 环绕主体：立体感、动感
        - 弧形运动：戏剧性、强调
        - 半环绕（180度）：对话、对峙

    跟拍类
        - 跟拍镜头：跟随主体移动
        - 背后跟拍：跟随主体背影
        - 侧面跟拍：平行跟随
        - 一镜到底：全程无剪辑的连续镜头

    升降类
        - 上升镜头：史诗感、希望
        - 下降镜头：压迫、沉重
        - 航拍镜头：规模、全景

    高级运镜
        - 希区柯克变焦：推拉+变焦的眩晕效果，表现心理冲击
        - 鱼眼镜头：超广角畸变，营造扭曲不安感
        - 低角度仰拍：低机位向上拍摄，增加英雄感/压迫感
        - 俯拍 / 鸟瞰：从高处向下拍摄，全局感
        - 第一人称主观视角：从角色视角出发，沉浸感
        - 快速摇镜：极速水平旋转产生运动模糊
        - 机械臂跟随：多角度灵活跟随人物视线

    手持与固定
        - 手持镜头：真实感、紧张
        - 固定机位：稳定、庄重

[动作词汇]

    面部
        - 缓慢睁眼、微笑浮现、目光转移、眉毛上扬
        - 表情从X转为Y、咬嘴唇、眼眶泛红

    头部
        - 转头、微微歪头、缓慢点头、抬头、低头

    身体
        - 呼吸起伏、前倾、后仰、站起、坐下
        - 向前走、伸出手、拥抱

    手部
        - 举手、握紧、手指动作、比划

    环境互动
        - 头发随风飘动、衣服飘动、水花飞溅
        - 落叶飘落、烟雾升起、尘埃飘浮

[速度修饰词]

    缓慢/轻柔：缓慢、轻柔、渐渐、微妙、细腻
    中等/自然：平稳、稳定、自然、流畅
    快速/强烈：快速、迅速、突然、戏剧性、爆发

[氛围与风格词汇]

    画面风格
        - 电影感、影片质感、大片风格、纪录片风格
        - 电影级质感，胶片颗粒，浅景深
        - 9:16竖屏电影感，24fps
        - 黑白水墨风格 / 动漫风格 / 超写实
        - 高饱和霓虹色调，冷暖对比

    情绪氛围
        - 戏剧性、紧张、悬疑、浪漫、梦幻、怀旧
        - 神秘、欢快、忧郁、暗黑奇幻、仙侠高燃
        - 喜剧风格，表情夸张
        - 温暖治愈、史诗恢宏

[音频设计]

    音频是 Seedance 2.0 的重要能力，提示词中应包含音频指导：

    背景音乐
        - 直接描述风格：背景音乐：恢宏大气 / 紧张悬疑 / 温暖治愈
        - 引用参考：背景BGM参考 @音频1
        - 引用视频中的音乐：音乐风格参考 @视频1

    音效设计
        - 环境音效：走路声、人群声、汽车声、雨声、风声
        - 动作音效：脚步声、呼吸声、衣料摩擦声
        - 情绪音效：心跳声、钟表声、回声

    音画同步
        - 转场画面与音乐节奏卡点
        - 脚步声、呼吸声必须与画面动作贴合
        - 音乐高潮与画面高潮同步

    旁白/对话
        - 角色对白直接写入提示词，用引号标注
        - 可指定音色参考：旁白音色参考 @视频1
        - 可指定语气：哽咽、怒吼、低语、坚定

[视频操作模式]

    除了标准的"从零生成"，Seedance 2.0 还支持以下操作模式，分镜师在需要时可以使用：

    视频延长（向后）
        将 @视频1 延长[时长]。
        [描述延长部分的画面内容]
        注意：生成时长应选择"新增部分"的时长（如延长5秒，生成长度也选5秒）

    视频延长（向前）

        - 从导演阐述中识别主要动作链
        - 保留具体物理动作，删除抽象描述

        - 如果导演未明确指定运镜，根据情绪和叙事目的补充

[禁忌规则]

    绝对禁忌
        - 否定句（"不要移动" → "保持静止"）
        - 重复参考图中已有的静态内容
        - 抽象概念（"体现自由" → 用具体动作表达）
        - 一条提示词中包含叙事上不相关的场景
        - @引用不指定用途（"参考@视频1" → "参考@视频1的运镜效果"）
        - 上传写实真人脸部素材

    避免事项
        - 过长提示词（控制信息密度与时长匹配）
        - 违反物理规律的动作
        - 与参考图内容矛盾的描述
        - 指令冲突（同时要求"固定镜头"和"环绕镜头"）
        - 单个连续镜头内节拍过密
        - 素材上传了但提示词中未用 @ 标注用途
        - 时长与内容密度不匹配
