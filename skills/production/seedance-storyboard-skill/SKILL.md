# Seedance 2.0 分镜编写技能

[技能说明]
    将导演的讲戏内容转化为 Seedance 2.0 格式的动态视频提示词。每个剧情点 = 一段完整叙事 = 一条提示词 = 一次 Seedance 2.0 生成任务。段落内可包含多组镜头（如蒙太奇、连续切换）。提示词中使用 @引用语法关联人物和场景素材，输出可直接复制到 Seedance 2.0 平台生成视频。

[文件结构]
    seedance-storyboard-skill/
    ├── SKILL.md                                    # 本文件（技能流程说明）
    ├── seedance-prompt-methodology.md              # Seedance 2.0 提示词方法论
    ├── examples/
    │   └── seedance-prompt-examples.md             # Seedance 2.0 官方提示词参考示例
    └── templates/
        └── seedance-prompts-template.md            # Seedance 提示词产出模板（输出格式定义）

[执行流程]

    第一步：读取上游产物
        - 读取导演讲戏本（01-director-analysis.md）
        - 读取人物提示词（assets/<剧名>/character-prompts.md）
        - 读取场景道具提示词（assets/<剧名>/scene-prompts.md）
        - 读取 seedance-prompt-methodology.md
        - 读取 examples/seedance-prompt-examples.md，了解官方提示词的风格和结构

    第二步：建立素材对应表
        - 根据人物和场景提示词文件，为本集涉及的素材分配 @引用编号
        - 只有在 assets/<剧名>/character-prompts.md 和 assets/<剧名>/scene-prompts.md 中有提示词的人物/场景才分配 @引用
        - 不在素材文件中的人物（群演、一次性配角）不进对应表，在提示词中直接用文字描述
        - 在文档头部建立对应表
        - 具体格式见 templates/seedance-prompts-template.md

        场景素材的编号规则（重要）：
        - assets/<剧名>/scene-prompts.md 中的九宫格是"生成格式"——用于一次性生成场景参考图
        - 生成后，用户会将九宫格中的每个格子单独提取出来，作为独立的场景参考图
        - 因此在素材对应表中，每个场景必须独立编号为一个 @图片，不能将整张九宫格作为一个 @图片
        - 编号顺序：先人物（按 assets/<剧名>/character-prompts.md 顺序），再场景（按 assets/<剧名>/scene-prompts.md 中的格子顺序）
        - 示例：如果有 4 个人物 + 9 个场景，则 @图片1-4 为人物，@图片5-13 为场景

        平台约束（单条提示词维度）：
        - 以下限制是针对每条提示词（每次 Seedance 生成任务），不是全文档限制
        - 图片 ≤ 9 张 / 条
        - 视频 ≤ 3 个 / 条
        - 音频 ≤ 3 个 / 条
        - 总文件数 ≤ 12 个 / 条
        - 素材对应表是全文档的总映射，可以超过单条限制

    第三步：为每个剧情点编写 Seedance 2.0 提示词
        从导演阐述中提取并转化：
        - 运镜：导演描述的镜头运动 → Seedance 运镜语言
        - 景别梯度：导演描述的规模镜头、人物近景/特写、结果落点镜头 → 可执行的镜头顺序
        - 动作：导演描述的人物动作 → 具体物理动作描述
        - 台词/声音：导演描述的对白 → 角色语音/旁白/环境音
        - 光影：导演描述的光影氛围 → 光线/色调/氛围描述
        - 节奏：导演描述的情绪节奏 → 运动速度/剪辑节奏
        - 场景层级：导演描述的主场、源头空间、后果空间 → 对应的场景 ref 选择与切换
        - 统一主时间线：先写唯一的复合 timeline，把镜头、对白、声音和尾帧交棒融合在同一组 beat 里，再由系统派生兼容字段

    第四步：输出
        按照 templates/seedance-prompts-template.md 定义的格式输出。

[Seedance 2.0 提示词写法核心规则]

    叙事描述式
        - 用完整的导演式段落描述，不要关键词堆叠
        - Seedance 2.0 擅长理解长段叙事性提示词
        - 参考官方模板风格：连续段落，包含运镜、动作、光影、声音的融合描述

    运镜描述
        - 用具体方向和动作描述镜头运动
        - "镜头从侧脸缓缓推向窗外夜色" ✓
        - "推镜头" ✗（太简略）
        - 常用运镜：推镜/拉镜、横摇/纵摇、环绕、跟拍、升降、手持

    景别梯度与电影重音
        - 高价值节点不要全程停留在同一景别，要主动组织“规模镜头 → 人物近景/特写 → 结果/后果镜头”
        - 宏大不是只靠全景。大景负责交代秩序、空间和压迫；近景/特写负责交代人物欲望、恐惧、代价和关系变化
        - 若一条分镜里既有群像调度又有人物命运转折，至少要给一拍看世界，一拍看人，一拍看结果，避免只有剧情还原、没有电影重音

    场景引用选择
        - 不要只因为当前人物站在主台中央，就把所有镜头都绑定局部主场；如果剧情涉及高台、四门、山头、阵地、废墟态、法阵外圈或其他宏大母体空间，必须把对应场景 ref 真正用起来
        - 多空间事件优先按“源头空间 / 主战空间 / 后果空间”组织引用，让 Seedance 知道规模从哪里来、冲击传到哪里、结果落在哪
        - 同一条提示词内的场景 ref 应服务镜头层次，而不是堆数量；优先保留最能建立规模、秩序和结果落点的 2-4 个关键场景锚点

    动作描述
        - 用具体的物理动作，避免抽象概念
        - "她揉了揉太阳穴，目光从屏幕移开，靠向椅背" ✓
        - "她感到疲惫" ✗

    时长控制
        - 遵循导演建议的时长
        - 底层规律：每个连续镜头内 1 拍 ≈ 2.5 秒屏幕时间，节拍太密动作会糊、跳、不自然
        - 多镜头段落：节拍分布在多个镜头切换中，每个镜头各自遵守密度约束，总节拍可超单镜头上限
        - 头尾安全区：每次 Seedance 生成前 0.5s 和后 0.5s 为安全区，不放关键内容

    声音/台词
        - 台词用引号标注
        - 可指定语气/声音特征
        - 可指定背景音乐风格
        - 可指定环境音
        - 同一条提示词中的完整台词必须串行出现，前一句结束后下一句再进入，不允许双声叠台词
        - 如果一条提示词里有多句对白，必须给出明确时间顺序或镜头顺序，优先使用“0.5-2秒……”这类表达
        - 如果对白过多导致时长拥挤，必须压缩到 1-3 句关键台词，并用反应镜头/静默/动作停顿隔开
        - 对白时间不应独立于镜头节拍存在；最稳的写法是把对白直接落在主时间线对应 beat 上

[群演/一次性配角处理]
    导演讲戏本中可能提到不在 assets/<剧名>/character-prompts.md 中的人物（群演、一次性配角）。处理方式：
    - 不为这些人物分配 @引用编号
    - 在提示词中直接用文字描述其外观和动作
    - 外观描述从导演讲戏本的导演阐述中提取（导演已在阐述中具体描述了这些角色的外观）
    - 示例：导演阐述中提到"路人甲侧头对同伴低声说"，讲戏本中描述其为"尖脸眯缝眼的白袍学员"
      → 提示词中写"一个尖脸眯缝眼的白袍学员侧头低声说..."，不使用 @引用

    连续性要求
        - 每条提示词都要明确承接上一段的尾态，并把情绪、动作或关系送到本段结束落点
        - 不允许把导演讲戏本中的关键对白链条、羞辱反击链条、揭示链条压缩成空泛概述
        - 必须拆出镜头节拍顺序，避免画面和台词同时过载
        - 每条提示词都要先明确当前在镜角色、入画路径和离场/被遮挡方式，避免人物凭空出现或消失
        - 运镜必须依附稳定空间锚点与明确路径，切镜由动作、视线、声响或特效冲击触发，避免跳轴和镜头漂移
        - 高能特效、法术、爆闪和机械齐鸣必须写成有源头、有传播、有余波、有环境反馈的物理事件，不要只给抽象效果词
        - 每个镜头节拍至少补 1-2 个可见可感的物理细节，如布料、石屑、金属反光、火星、焦痕、脚步或呼吸
        - 群像对峙、翻盘、奇观兑现和大场面压制段，至少要交代一拍规模镜头、一拍人物反应近景/特写和一拍结果/后果落点，避免整条都停在局部表演区

[禁忌规则]
    - 不要使用否定句（AI 不理解否定，可能产生相反效果）
        "不要移动镜头" ✗ → "镜头保持固定" ✓
    - 不要重复导演讲戏本和人物/场景提示词中已有的静态内容描述
        重点描述"变化"：动作、运动、情绪转变
    - 不要在一条提示词中塞入叙事上不相关的场景
        一条提示词 = 一个完整的叙事段落（可含多组镜头，但叙事连续）
    - 不要使用与导演讲戏本和人物/场景提示词矛盾的描述
    - 不要为不在 assets/<剧名>/character-prompts.md 中的人物使用 @引用

[注意事项]
    - seedance-prompt-methodology.md 是核心方法论，必须遵守
    - 参考 examples/seedance-prompt-examples.md 了解官方推荐的提示词风格和结构
