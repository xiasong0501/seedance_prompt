# 爆款编剧技能

[技能说明]
    该技能面向“同一集有多个模型版本剧本”的场景，目标不是重写故事主线，而是在保留核心剧情的前提下，提高剧本的抓眼球程度、短时留存和传播性。

    适用输入：
    - script/<剧名>/ 下的同集多个剧本版本
    - 可选：analysis/<剧名>/explosive-actor-gpt/series_hit_playbook.md（本剧专属爆款经验）

    产出：
    1. 单集爆款评分报告
    2. 单集改稿策略
    3. 强化后的爆款版剧本
    4. 整剧层面的爆款玩法手册

[核心原则]
    - 保故事，不保平：主线逻辑要稳，但表达必须更有抓力
    - 强钩子优先：前3秒必须有异常、反差、危险、羞耻、秘密、利益冲突中的至少一种
    - 动机外显：人物为什么这么做，要更快、更狠、更具体地让观众感知到
    - 冲突递进：每一段不是“信息补充”，而是“局势变坏 / 欲望更急 / 代价更大”
    - 情绪兑现：铺垫过的委屈、恩情、羞耻、心动、危险，必须在关键节点得到足够强的情绪释放
    - 结尾卡点：每集末尾必须留下“马上想点下一集”的问题或局势变化

[执行流程]

    第一步：收集同集多个版本
        - 读取同一集的多个模型版本剧本
        - 明确每个版本的来源（openai / gemini / 其他）
        - 如存在本剧专属爆款手册，先读取并吸收

    第二步：逐版本打分
        从以下维度给每个版本评分：
        - 开篇钩子：前3秒和前30秒是否足够抓人
        - 冲突强度：局势是否迅速进入矛盾核心
        - 情绪拉力：恩情、羞耻、危险、反差、欲望是否够强
        - 节奏效率：是否有拖沓解释、重复信息、平铺叙述
        - 人物欲望：主角和关键角色是否“有想要的东西”
        - 结尾卡点：是否能强力推动下一集点击

    第三步：提炼有效爆点
        对比多个版本后，筛出最有效的内容：
        - 哪个版本的开头更勾人
        - 哪个版本的旁白/台词更有传播性
        - 哪个版本的人物关系更有情绪张力
        - 哪个版本的羞耻感、危险感、反差感更强
        - 哪些表述虽然热闹，但会削弱人物可信度或触发合规风险

    第四步：制定改稿蓝图
        改稿时明确：
        - 必须保留的故事节点
        - 必须放大的高光桥段
        - 应该压缩或删掉的解释段
        - 需要前置的冲突与信息
        - 必须强化的卡点与金句

    第五步：输出强化版剧本
        输出标准：
        - 保留剧本可被 director-skill 继续处理的结构
        - 分场明确，信息清楚
        - 可读性高，旁白和台词更有锋利度
        - 强调人物处境、代价、危险、情绪和反差
        - 结尾必须更钩人

[爆款改稿重点]
    开头：
    - 先上最危险、最暧昧、最反常或最羞耻的画面
    - 信息不必一次讲全，但情绪必须先钩住

    中段：
    - 每个段落都要回答“这段为什么观众不会划走”
    - 避免纯说明型独白，尽量让信息伴随动作、冲突或情绪出现

    人物：
    - 女主要让观众想“她太惨了 / 太敢了 / 太能忍了 / 太值了”
    - 男主要让观众想“他现在这么惨 / 以后一定会反转”
    - 配角要承担施压、推动、反差或情绪放大的功能

    结尾：
    - 用“刚做出决定 / 刚被看穿 / 刚要出手 / 刚发生反转”的临界点收住
    - 不要用情绪已经落地后的平场景做结尾

[本剧专属经验沉淀]
    如果 analysis/<剧名>/explosive-actor-gpt/series_hit_playbook.md 存在：
    - 优先读取该文件
    - 把其中沉淀出的钩子模板、卡点规律、人物高光打法，继续用于后续集数
    - 如果新剧本出现更好的爆点，也要反向补充到新的玩法手册中
